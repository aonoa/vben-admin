# Iteration Template

1. Read project docs and current Git state in both repos.
2. Confirm the active version line, task boundary, paired backend impact, and write paths.
3. Implement the change without touching unrelated dirty files.
4. Verify with the narrowest meaningful frontend command set plus paired backend verification when the user flow can be affected.
5. Update project docs if the work changed decisions or boundaries.
6. Report changed files, frontend/backend impact, verification, remaining risks, and the next action.
